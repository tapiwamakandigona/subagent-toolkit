# Artifact: Architecture

The required skeleton for an architecture document — the output of the Architecture phase (usually `agents/architect.md`) and the design authority for every build phase after it. Its interface contracts get pinned verbatim into builder briefs; its module map is the orchestrator's ownership map.

## Usage notes

- Interface contracts are exact code/schema blocks, written to be copied — never prose descriptions of an interface.
- The design freeze is what makes parallel building safe: builders implement against these contracts and do not change them unilaterally; contract problems route back to the architect, who amends for everyone at once.
- The generated/shared-file list is the integrator's instruction set: these files are regenerated by command, never hand-merged.
- Keep rationale honest: each stack choice names the requirement that drove it and the strongest rejected alternative.

## Template

```text
# Architecture: {{project_name}}
Spec: {{path_to_spec}}   Status: {{draft | frozen}}   Date: {{date}}

## Stack
{{choice}}: {{driving_requirement}} — over {{rejected_alternative}} because {{reason}}
{{one_line_per_major_choice}}

## Module map
| Module | Responsibility | Owner (one at a time) | Paths |
|---|---|---|---|
| {{name}} | {{one_line}} | {{role_or_agent_slot}} | {{path_globs}} |

## Interface contracts
### {{contract_name}} ({{modules_it_connects}})
    {{exact_signatures_routes_or_schemas — a fenced code block in the real
    document, copy-paste precision, one block per cross-module seam}}

## Data model
{{entities_fields_relationships — and which module owns each migration}}

## Design freeze
Builders implement against the contracts above and MUST NOT change them
unilaterally. If a contract is wrong or incomplete: stop, report to the
orchestrator/architect; the amendment is published here, then work resumes.

## Generated & shared files
| File | Regenerated by | Never hand-edited because |
|---|---|---|
| {{lockfile_or_generated_path}} | {{exact_command}} | {{reason}} |

## Requirement coverage
{{FR_id}} → {{module(s)_that_deliver_it}}   ({{every_spec_requirement_has_a_home}})
```

## Anti-patterns

- **Prose interfaces** — "the API module exposes user operations" cannot be coded against; the signature block can.
- **Shared-boundary modules** — a file two owners both need to edit is a boundary drawn wrong; redraw before building starts.
- **Resume-driven stack choices** — technology without a driving requirement is risk without payoff.
- **Silent contract drift** — a builder "improving" a frozen interface breaks every sibling; the freeze section exists to make that a reportable event, not a judgment call.
